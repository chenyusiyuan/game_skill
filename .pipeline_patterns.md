# Pipeline Failure Patterns

跨 case 的失败模式记录。`phase_gate_reviewer` 会读这份文件判定 Blocker 是 CASE-LOCAL 还是 SYSTEMATIC。

## 规则

- 每条 CASE-LOCAL Blocker 修完后必须追加一行，累计 +1
- 同一 pattern 累计 3 次 → 下次再命中自动升为 SYSTEMATIC
- 当 SYSTEMATIC 修复后，对应 pattern 行末标注"**抽象升级 @ commit `<hash>`**"
- 行格式固定，不要换列序
