# Brief: pixel_pixijs

## Raw Query
像素传送带清板益智游戏开发 Prompt

1. 核心定义
- 游戏类型：像素风、轻度休闲、益智策略、竖屏点击游戏。
- 核心逻辑：四周环形传送带、颜色匹配攻击、有限弹药单位、等待槽回收、容量管理、顺序清板。
- 核心体验：玩家点击不同颜色的小猪进入棋盘四周的传送带，小猪沿外圈移动，只能根据自己当前所处的位置，依序处理正对方向上的目标块，而不是全屏自动索敌。玩家需要通过合理安排出场顺序、颜色搭配和等待槽管理，逐步清空整块像素区域。

2. 场景结构
游戏主要由三个核心区域构成：
- 中央棋盘： 位于屏幕中央，由不同颜色的像素块组成，是玩家需要清空的目标区域。
- 四周传送带： 位于棋盘外围，是一圈闭环轨道，小猪只能沿着棋盘外圈移动。
- 底部等待槽： 位于屏幕下方，用于放置待命或回收的小猪，玩家点击后可再次派出。

场景特征：
- 中央棋盘是固定网格，例如 6×6 或 7×7。
- 棋盘中的方块不会下落、不会补充、不会重排。
- 目标是将棋盘上的所有目标块逐步清空。

3. 小猪单位系统
每只小猪至少包含两个核心属性：
- 颜色： 如红、蓝、黄、绿等。
- 数值： 表示该小猪当前剩余可攻击次数，也就是剩余弹药量。

单位规则：
- 玩家点击底部的一只小猪后，它会进入棋盘四周的传送带。
- 小猪进入传送带后自动沿外圈持续移动。
- 传送带上可以同时存在多只小猪，不是严格一次只能上一只。
- 传送带有容量上限，超过容量后不能继续派出新的小猪，必须等待空位出现。
- 小猪数值耗尽后直接离场消失。
- 若小猪离开传送带时仍有剩余数值，则进入底部等待槽，保留剩余数值，后续可以再次点击派出。

4. 攻击与索敌规则
攻击规则必须严格依赖"小猪当前所在位置"，而不是全屏自动寻找同色块：
- 当小猪移动到棋盘上边时，只能检查自己正对那一列中"从上往下数的第一个未消除目标块"。
- 当小猪移动到棋盘右边时，只能检查自己正对那一行中"从右往左数的第一个未消除目标块"。
- 当小猪移动到棋盘下边时，只能检查自己正对那一列中"从下往上数的第一个未消除目标块"。
- 当小猪移动到棋盘左边时，只能检查自己正对那一行中"从左往右数的第一个未消除目标块"。

命中规则：
- 只有当这个"当前方向上的第一个目标块"与小猪颜色相同时，小猪才会攻击一次。
- 每次攻击只消耗 1 点数值。
- 每次攻击只处理 1 个目标块。
- 如果当前方向上的第一个目标块颜色不匹配，则本次经过不攻击，直接继续前进。
- 小猪不能跳过前方异色块去攻击后方同色块。
- 小猪不能一次攻击整行、整列或全屏所有同色块。

5. 目标块系统
中央棋盘中的每个格子都是一个目标块，可包含以下类型：
- 普通块： 耐久为 1，被对应颜色命中 1 次后立刻消失。
- 强化块： 耐久为 2 或更高，需要被对应颜色命中多次后才会消失。
- 锁定块： 必须先清除外层阻挡后，内侧目标才会被轨道上的小猪逐步处理。
- 空白格： 已被消除的位置，不再参与阻挡和判定。

规则要求：
- 目标块消失后，该位置变为空，不发生下落和填充。
- 因为采用"当前方向第一个目标块"的判定机制，所以外层块会天然形成阻挡关系。
- 玩家需要先处理外围阻挡块，后续同色小猪才能逐步攻击到更内层的目标块。

6. 等待槽与循环机制
等待槽是玩法核心之一，不只是普通备战栏：
- 底部存在固定数量的等待槽，例如 5 个。
- 尚未派出的小猪初始放在等待槽中。
- 小猪离开传送带时，如果还有剩余数值，则进入等待槽并保留数值。
- 玩家可以根据当前棋盘局势，再次点击等待槽中的小猪，让其重新进入外圈传送带。
- 这形成核心循环：派出 → 沿四周移动 → 依序攻击 → 数值未耗尽则回收 → 再次派出。

策略意义：
- 玩家不是一次性消耗所有小猪，而是在有限容量和等待槽约束下，反复调度同一批单位。
- 某些高数值小猪适合保留到更合适的位置和时机再上场。
- 等待槽和传送带容量共同决定了节奏与资源管理深度。

7. 关卡节奏与策略重点
关卡节奏建议逐步递进：
- 前期关卡： 2 种颜色，帮助玩家理解四周传送带和"当前方向第一个目标块"的规则。
- 中期关卡： 3 种颜色，开始强调阻挡关系、颜色顺序与小猪回收利用。
- 后期关卡： 4 种颜色，并加入强化块或局部锁定结构，提升策略深度。

核心策略点：
- 玩家需要决定先派出哪种颜色的小猪。
- 玩家需要判断哪些外层异色块必须优先清掉，否则会持续挡住后面的关键目标。
- 玩家需要避免高数值小猪在不匹配的位置上大量空转，浪费弹药。
- 玩家需要利用等待槽回收未耗尽的小猪，在后续更有利的时机重新派出。
- 正确顺序可以通关，错误顺序会导致资源耗尽后棋盘仍未清空。

8. 胜负规则
- 胜利条件： 清空中央棋盘中的全部目标块。
- 失败条件： 所有小猪都已耗尽离场，或剩余等待槽中的小猪已无法继续有效清除棋盘中的剩余方块。
- 游戏应体现明显的"顺序决策"属性，而不是无脑点击即可过关。
- 每一关都应让玩家感受到：只要调整派出顺序，局面就可能从失败变为通关。

9. 战斗反馈（Game Feel）
- 视觉： 小猪在外圈移动时路径清晰可见，攻击时发射小球或像素弹。
- 命中： 目标块被正确颜色命中时闪烁、缩放或掉数字。
- 消除： 目标块耐久归零后立即消失，并留下空格。
- 回收： 小猪带着剩余数值返回等待槽时，要有明显的回收动画和数值保留反馈。
- UI： 顶部显示关卡信息，中部显示中央棋盘与四周传送带，底部显示等待槽、小猪颜色和剩余数值。
- 操作： 以点击派出为主，整体体验简洁直观，易上手但具备清晰的策略深度。

10. 强约束
- 禁止做成全屏自动索敌。
- 禁止小猪一次攻击多个同色块。
- 禁止穿透前方异色块攻击后方目标。
- 禁止做成整行整列直接清除。
- 禁止做成三消、塔防、自动射击或普通消除游戏。
- 必须严格体现"四周传送带 + 按当前位置依序处理当前方向上的第一个目标块 + 等待槽回收再派出"的玩法核心。
