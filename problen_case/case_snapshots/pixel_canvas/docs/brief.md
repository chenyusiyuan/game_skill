# Brief: pixel_canvas

## Raw Query
像素传送带清板益智游戏开发 Prompt

1. 核心定义
- 游戏类型：像素风、轻度休闲、益智策略、竖屏点击游戏。
- 核心逻辑：四周环形传送带、颜色匹配攻击、有限弹药单位、等待槽回收、容量管理、顺序清板。
- 核心体验：玩家点击不同颜色的小猪进入棋盘四周的传送带，小猪沿外圈移动，只能根据自己当前所处的位置，依序处理正对方向上的目标块，而不是全屏自动索敌。玩家需要通过合理安排出场顺序、颜色搭配和等待槽管理，逐步清空整块像素区域。

2. 场景结构
- 中央棋盘：固定网格（6×6 或 7×7），方块不会下落、补充、重排
- 四周传送带：闭环轨道，小猪沿外圈移动
- 底部等待槽：待命或回收的小猪

3. 小猪单位系统
- 颜色：红、蓝、黄、绿
- 数值：剩余弹药量
- 规则：点击进入传送带，自动移动，数值耗尽消失，未耗尽回等待槽

4. 攻击与索敌规则（核心）
- 位置依赖：只能攻击当前位置正对方向的第一个目标块
- 颜色匹配：只有颜色相同才攻击
- 单次攻击：每次消耗1点数值，处理1个目标块
- 禁止穿透：不能跳过异色块攻击后方目标

5. 目标块系统
- 普通块：耐久1
- 强化块：耐久2+
- 锁定块：需先清除外层阻挡
- 空白格：消除后的空位

6. 等待槽与循环机制
- 固定数量等待槽（如5个）
- 回收未耗尽小猪，保留数值
- 核心循环：派出 → 移动 → 攻击 → 回收 → 再派出

7. 关卡节奏
- 前期：2种颜色
- 中期：3种颜色
- 后期：4种颜色 + 强化块

8. 胜负规则
- 胜利：清空全部目标块
- 失败：小猪耗尽无法清空

9. 强约束（必须严格遵守）
- 禁止全屏自动索敌
- 禁止一次攻击多个同色块
- 禁止穿透前方异色块攻击后方目标
- 禁止整行整列直接清除
- 禁止做成三消、塔防、自动射击
- 必须严格体现"四周传送带 + 按位置依序处理第一个目标块 + 等待槽回收"
